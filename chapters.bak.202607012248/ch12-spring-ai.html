Java 生态的 Agent 框架

12.1 Java 生态的 Agent 之路

Agent 框架大多集中在 Python 生态（LangChain、LangGraph、AutoGen...）。但企业级 Java 项目怎么办？Spring AI 填补了这个空白。

💡 Spring AI 的定位

Spring AI 是 Spring 生态的 AI 集成框架。它不是 reinvent the wheel，而是把 Spring 的成熟工程实践（依赖注入、配置管理、声明式编程）带入 AI 开发。让 Java 开发者用熟悉的方式构建 Agent。

12.3 Spring AI 代码实战

Java: Spring AI 构建 ReactAgent 
// ===== 1. 配置 ChatClient =====
@Configuration
public class AIConfig {
    
    @Bean
    public ChatClient chatClient(ChatClient.Builder builder) {
        return builder
            .defaultSystem("你是一个智能助手")
            .defaultTools(weatherTool(), searchTool())
            .build();
    }
}

// ===== 2. 定义工具 =====
@Component
public class WeatherTool {
    
    @Tool(description = "获取指定城市的天气")
    public String getWeather(String city) {
        return weatherService.query(city);
    }
}

@Component  
public class SearchTool {
    
    @Tool(description = "搜索互联网获取信息")
    public String search(String query) {
        return webSearchService.search(query);
    }
}

// ===== 3. 构建 ReactAgent =====
@Service
public class AgentService {
    
    @Autowired
    private ChatClient chatClient;
    
    public String ask(String question) {
        // ReactAgent = LLM + 工具 + 循环
        return chatClient.prompt()
            .user(question)
            .call()
            .content();
    }
}

// ===== 4. Graph 工作流（类似 LangGraph）=====
@Bean
public StateGraph researchWorkflow() {
    return new StateGraph<>(AgentState.class)
        .addNode("planner", plannerNode())
        .addNode("searcher", searcherNode())
        .addNode("writer", writerNode())
        .addEdge(START, "planner")
        .addConditionalEdge("planner", routeByPlan())  // 条件边
        .addEdge("searcher", "writer")
        .addEdge("writer", END)
        .compile();
}

Spring AI Alibaba 扩展

Spring AI Alibaba 是阿里巴巴在 Spring AI 基础上的扩展，增加了通义千问模型集成、企业级特性：

通义千问集成

原生集成 DashScope（通义千问 API），支持 qwen-max/turbo/plus 等模型。

检查点恢复

Graph Runtime 支持检查点持久化，故障后可从断点恢复执行。

A2A 协议支持

支持 Agent-to-Agent 通信协议，跨框架 Agent 协作。

MCP 集成

1.1/2.0 版本原生集成 MCP 协议，直接使用 MCP Server 工具。

12.4 Spring AI 的多 Agent 编排

Java: 多 Agent 编排示例 
// 定义多个专业 Agent
@Bean
public Agent researcherAgent(ChatClient chatClient) {
    return ReactAgent.builder()
        .name("researcher")
        .instruction("你是研究员，负责信息搜集")
        .tools(searchTool())
        .chatClient(chatClient)
        .build();
}

@Bean
public Agent writerAgent(ChatClient chatClient) {
    return ReactAgent.builder()
        .name("writer")
        .instruction("你是技术写手，负责撰写报告")
        .chatClient(chatClient)
        .build();
}

// 编排多 Agent 协作
@Bean
public StateGraph multiAgentWorkflow(
        Agent researcherAgent, 
        Agent writerAgent) {
    
    return new StateGraph<>(AgentState.class)
        .addNode("research", researcherAgent)
        .addNode("write", writerAgent)
        .addNode("review", reviewNode())
        .addEdge(START, "research")
        .addEdge("research", "write")
        .addEdge("write", "review")
        .addConditionalEdge("review", needRevise())  // 需要修改？
        .compile(checkpointer);  // 带检查点
}

12.5 Spring AI 的企业级特性

🏭 为什么企业选择 Spring AI
Spring 生态融合

无缝集成 Spring Boot、Spring Cloud、Spring Security。已有 Spring 项目可直接加入 AI 能力。

声明式编程

@Tool 注解定义工具，@Bean 注入 Agent。和写 Spring Controller 一样简单。

可观测性

集成 Micrometer、Spring Boot Actuator。Agent 调用链路可监控、可追踪。

配置管理

application.yml 统一配置模型、工具、知识库。支持 Profile 环境隔离。

📋 八股总结 — 面试高频考点
Q1: Spring AI 的三层架构是什么？各自职责？
① Augmented LLM（增强LLM层）：LLM + 工具 + 记忆。核心组件：ChatClient、ToolCallback、Memory。
② Graph Runtime（图运行时）：工作流编排引擎。核心组件：StateGraph、Node、Edge、Checkpoint。类似 LangGraph。
③ Agent Framework（Agent框架层）：高级 Agent 模式。核心组件：ReactAgent、MultiAgent、A2A 协议。

从下到上：模型能力 → 流程编排 → 高级模式。
Q2: Spring AI 和 LangGraph 的 Graph Runtime 有什么相似和不同？
相似：都用图结构定义工作流——StateGraph + Node + Edge + Conditional Edge + Checkpoint。概念几乎一一对应。
不同：
① 语言：Spring AI 是 Java，LangGraph 是 Python
② 生态：Spring AI 融合 Spring 生态（DI、配置、安全），LangGraph 融合 Python AI 生态
③ 编程范式：Spring AI 声明式（注解+Bean），LangGraph 命令式（函数式API）
Q3: Spring AI Alibaba 相比 Spring AI 增加了什么？
① 通义千问集成：原生支持 DashScope API（qwen-max/turbo/plus）
② 检查点恢复：Graph Runtime 支持持久化检查点，故障后从断点恢复
③ A2A 协议：Agent-to-Agent 跨框架通信
④ MCP 集成：1.1/2.0 版本原生支持 MCP 协议
⑤ 企业级特性：适配阿里云生态（OSS、RDS、SLS等）
Q4: 什么场景下应该选择 Spring AI 而不是 LangGraph？
适合 Spring AI 的场景：
① 已有 Spring Boot 项目，想加入 AI 能力
② 团队是 Java 技术栈，没有 Python 工程师
③ 需要企业级可观测性（Micrometer/Actuator）
④ 需要和现有 Spring Security/Cloud 集成
⑤ 对类型安全和编译期检查有要求

适合 LangGraph 的场景：Python 生态、需要最新 AI 特性、研究性项目。
Q5: Spring AI 的 @Tool 注解和 Python 的 Function Calling 有什么区别？
本质相同：都是定义工具让 LLM 调用。

区别：
① Spring AI 用 @Tool 注解标记方法，自动生成 JSON Schema。Python 需要手动写 Schema 字典。
② Spring AI 的工具是 Spring Bean，可以注入任何依赖（数据库连接、HTTP客户端等）。Python 工具是普通函数。
③ Spring AI 工具有 Spring 的 AOP/事务/安全等能力。Python 没有这些。
