第9章 AutoGen 与 CrewAI

两种风格迥异的多 Agent 框架

9.1 AutoGen：像群聊一样协作

微软推出的 AutoGen 把多 Agent 协作变成了"群聊"——Agent 之间通过消息自由对话，自主决定谁说话、说什么。

💡 AutoGen 的核心洞察

人类团队通过对话协作——有人提问、有人回答、有人补充。AutoGen 让 Agent 也这样工作：每个 Agent 有角色和职责，通过消息对话推进任务，框架自动管理对话轮次和终止条件。

AutoGen 的 Agent 类型

AssistantAgent

通用 AI 助手。有 System Prompt 定义角色，可调用 LLM。是团队中的"专家"。

UserProxyAgent

人类代理。可以执行代码、调用工具、或把消息转发给真实人类。是"执行者"。

GroupChatManager

群聊管理器。管理多个 Agent 的对话顺序、轮次限制和终止条件。是"主持人"。

ConversableAgent

可对话 Agent 的基类。支持自定义对话逻辑、工具调用和回复条件。

AutoGen 代码示例

Python: AutoGen 多 Agent 协作 
import autogen

# 配置 LLM
config_list = [{"model": "gpt-4o", "api_key": "sk-xxx"}]

# 1. 创建代码编写 Agent
coder = autogen.AssistantAgent(
    name="coder",
    system_message="""你是一个 Python 开发者。
    你负责编写代码解决用户问题。
    写完代码后交给 reviewer 审查。""",
    llm_config={"config_list": config_list}
)

# 2. 创建代码审查 Agent
reviewer = autogen.AssistantAgent(
    name="reviewer",
    system_message="""你是一个代码审查员。
    检查代码的正确性、安全性和效率。
    如果有问题，指出并让 coder 修改。
    如果没问题，说 'APPROVED'。""",
    llm_config={"config_list": config_list}
)

# 3. 创建用户代理（执行代码）
user_proxy = autogen.UserProxyAgent(
    name="user_proxy",
    human_input_mode="NEVER",  # 不需要人工输入
    code_execution_config={"work_dir": "coding"}
)

# 4. 创建群聊
groupchat = autogen.GroupChat(
    agents=[user_proxy, coder, reviewer],
    messages=[],
    max_round=10  # 最多 10 轮对话
)
manager = autogen.GroupChatManager(groupchat=groupchat)

# 5. 开始对话
user_proxy.initiate_chat(
    manager,
    message="写一个函数，输入一个URL列表，"
            "并发请求并返回状态码。"
)

运行后你会看到 Agent 之间的对话：

Agent 对话过程 
user_proxy: 写一个函数，输入一个URL列表，并发请求并返回状态码。

coder: 我来写一个使用 aiohttp 的并发请求函数：
```python
import aiohttp
import asyncio

async def check_urls(urls):
    async with aiohttp.ClientSession() as session:
        tasks = [check_one(session, url) for url in urls]
        return await asyncio.gather(*tasks)
...
```

reviewer: 代码有几个问题：
1. 没有超时设置，可能卡死
2. 没有异常处理，单个URL失败会影响全部
3. 建议加 timeout 和 try-except

coder: 好的，我修改一下：
```python
async def check_one(session, url, timeout=10):
    try:
        async with session.get(url, timeout=timeout) as resp:
            return url, resp.status
    except Exception as e:
        return url, str(e)
```

reviewer: APPROVED ✓

user_proxy: [执行代码... 结果: 成功]

9.2 CrewAI：像组建项目团队

CrewAI 的设计哲学完全不同：它把多 Agent 协作建模为"项目团队"——每个 Agent 有明确角色（Role）、目标（Goal）和背景（Backstory），任务（Task）按流程分配。

AutoGen 风格
群聊对话，自由协作
Agent 之间自由对话
灵活但不太可控
适合探索性任务
≈ 头脑风暴会议
CrewAI 风格
角色+任务，结构化协作
按流程分配任务
可控但不够灵活
适合标准化任务
≈ 项目团队分工

CrewAI 的核心概念

Agent（角色）

Role: 研究员
Goal: 找到最准确的信息
Backstory: 10年研究经验...
Tools: [search, analyze]

Task（任务）

Description: 调研XX框架
Expected Output: 一份报告
Assigned to: researcher
Context: [前序任务]

Crew（团队）

Agents: [研究员, 写手, 审核员]
Tasks: [调研, 写报告, 审核]
Process: sequential / hierarchical
Verbose: true

CrewAI 代码示例

Python: CrewAI 组建研究团队 
from crewai import Agent, Task, Crew, Process

# 1. 定义 Agent（角色）
researcher = Agent(
    role='资深研究员',
    goal='找到最准确、最全面的技术信息',
    backstory='10年技术研究员经验，擅长深度调研',
    tools=[search_tool, web_scraper],
    verbose=True
)

writer = Agent(
    role='技术写手',
    goal='把复杂技术信息写成易懂的报告',
    backstory='资深技术博客作者，擅长把复杂概念简单化',
    verbose=True
)

reviewer = Agent(
    role='质量审核员',
    goal='确保报告准确、完整、易读',
    backstory='前技术编辑，对细节要求极高',
    verbose=True
)

# 2. 定义 Task（任务）
research_task = Task(
    description='调研 LangGraph 框架的架构、'
                '核心概念和使用场景',
    expected_output='一份包含架构图、'
                    '代码示例和对比分析的调研报告',
    agent=researcher
)

write_task = Task(
    description='基于调研结果撰写一份'
                '3000字的技术分析报告',
    expected_output='一份结构清晰、'
                    '通俗易懂的技术报告',
    agent=writer,
    context=[research_task]  # 依赖调研任务的结果
)

review_task = Task(
    description='审核报告的准确性、'
                '完整性和可读性',
    expected_output='审核通过的报告'
                    '或修改建议',
    agent=reviewer,
    context=[write_task]
)

# 3. 组建 Crew（团队）
crew = Crew(
    agents=[researcher, writer, reviewer],
    tasks=[research_task, write_task, review_task],
    process=Process.sequential,  # 串行执行
    verbose=True
)

# 4. 启动
result = crew.kickoff()

9.4 AutoGen 完整实战代码

上一节我们看了 AutoGen 的基本用法，现在来一个更完整的实战——涵盖 UserProxyAgent + AssistantAgent 的双 Agent 协作、GroupChat 多 Agent 群聊、以及 Human-in-the-loop 人工介入三种核心模式。这些代码可以直接复制运行，是快速上手 AutoGen 的最佳起点。

模式一：UserProxyAgent + AssistantAgent 双 Agent 协作

这是 AutoGen 最基础也最常用的模式：UserProxyAgent 负责执行代码和与人类交互，AssistantAgent 负责生成解决方案和编写代码。两者通过消息对话推进任务，UserProxyAgent 可以自动执行 AssistantAgent 写出的代码并把结果反馈回去，形成"写代码→执行→反馈→改进"的闭环。下面是完整的双 Agent 协作代码示例：

Python: AutoGen 双 Agent 协作完整代码 
from autogen import AssistantAgent, UserProxyAgent

# 配置 LLM
llm_config = {
    "model": "gpt-4",
    "api_key": "sk-xxx",
    "temperature": 0.7
}

# 创建 AssistantAgent —— 负责生成代码和解决方案
assistant = AssistantAgent(
    name="assistant",
    llm_config=llm_config,
    system_message="You are a helpful AI assistant. "
                   "Write Python code to solve problems. "
                   "End with 'TERMINATE' when the task is done."
)

# 创建 UserProxyAgent —— 负责执行代码和与人类交互
user_proxy = UserProxyAgent(
    name="user_proxy",
    human_input_mode="TERMINATE",
    code_execution_config={"work_dir": "coding"},
    is_termination_msg=lambda x: x.get("content", "").rstrip().endswith("TERMINATE")
)

# 启动对话
user_proxy.initiate_chat(
    assistant,
    message="写一个 Python 脚本，抓取指定网页的标题和所有链接。"
)

模式二：GroupChat 多 Agent 协作

当任务需要多个不同角色的 Agent 协作时，GroupChat 模式就派上用场了。GroupChatManager 作为"主持人"管理所有 Agent 的发言顺序，确保对话有序进行。下面展示一个研究员 + 写手 + 审核员的三 Agent 协作场景，Manager 会根据对话内容自动调度最合适的 Agent 发言：

Python: AutoGen GroupChat 多 Agent 协作 
from autogen import AssistantAgent, UserProxyAgent, GroupChat, GroupChatManager

llm_config = {"model": "gpt-4", "api_key": "sk-xxx"}

# 研究员 Agent：负责搜集信息
researcher = AssistantAgent(
    name="researcher",
    llm_config=llm_config,
    system_message="You are a researcher. Find information "
                   "and share key findings. End with 'TERMINATE' when done."
)

# 写手 Agent：负责撰写报告
writer = AssistantAgent(
    name="writer",
    llm_config=llm_config,
    system_message="You are a writer. Write reports based "
                   "on research findings. End with 'TERMINATE' when done."
)

# 用户代理：发起任务
user_proxy = UserProxyAgent(
    name="user_proxy",
    human_input_mode="NEVER",
    code_execution_config={"work_dir": "coding"}
)

# 创建群聊：3 个 Agent，最多 10 轮对话
groupchat = GroupChat(
    agents=[user_proxy, researcher, writer],
    messages=[],
    max_round=10
)
manager = GroupChatManager(groupchat=groupchat, llm_config=llm_config)

# 启动群聊
user_proxy.initiate_chat(
    manager,
    message="Research AI Agent trends in 2026 and write a summary."
)

模式三：Human-in-the-loop 人工介入

在需要人类审核的关键节点，AutoGen 可以通过 human_input_mode 参数实现人工介入。设置为"ALWAYS"时，每轮对话后都会暂停等待人类输入；设置为"TERMINATE"时，仅在终止前确认。这种机制让人类可以在关键决策点进行干预和引导，特别适合高风险场景下的数据分析任务。

Python: AutoGen Human-in-the-loop 模式 
from autogen import AssistantAgent, UserProxyAgent

llm_config = {"model": "gpt-4", "api_key": "sk-xxx"}

# 创建需要人工确认的 UserProxyAgent
user_proxy = UserProxyAgent(
    name="user_proxy",
    human_input_mode="ALWAYS",  # 每轮都等待人类输入
    code_execution_config={"work_dir": "coding"},
    is_termination_msg=lambda x: "TERMINATE" in x.get("content", "")
)

assistant = AssistantAgent(
    name="assistant",
    llm_config=llm_config,
    system_message="You are a data analyst. "
                   "Analyze data and propose insights. "
                   "Wait for human feedback before proceeding."
)

# 启动对话 —— 每轮都会暂停等待人类输入
user_proxy.initiate_chat(
    assistant,
    message="分析以下销售数据趋势，给出下季度建议："
            "Q1: 120万, Q2: 150万, Q3: 180万, Q4: 210万"
)

9.5 CrewAI 完整实战代码

CrewAI 的实战代码展示了它独特的角色化分工模式。与 AutoGen 的自由对话不同，CrewAI 通过 Agent + Task + Crew 三层结构实现结构化协作。下面展示完整的顺序执行流程、自定义工具开发、以及层级执行模式的代码示例，这些代码覆盖了 CrewAI 最核心的三种使用场景。

模式一：Agent + Task + Crew 顺序执行

这是 CrewAI 最核心的用法：定义角色化的 Agent、分配具体 Task、组建 Crew 按顺序执行。每个 Task 有明确的 expected_output，前序任务的输出自动作为后序任务的上下文，形成清晰的工作流水线。下面的代码展示了一个完整的研究团队从调研到写作的顺序执行流程：

Python: CrewAI 顺序执行完整代码 
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool, ScrapeWebsiteTool

# 初始化工具
search_tool = SerperDevTool()
scrape_tool = ScrapeWebsiteTool()

# 定义 Agent —— 每个角色都有明确的 role/goal/backstory
researcher = Agent(
    role="Senior Research Analyst",
    goal="Uncover cutting-edge developments in AI Agent frameworks",
    backstory="You are an expert analyst with 10 years experience "
              "and a keen eye for emerging trends.",
    tools=[search_tool, scrape_tool],
    verbose=True
)

writer = Agent(
    role="Tech Content Writer",
    goal="Create engaging, accurate content about AI trends",
    backstory="You are a senior tech writer who simplifies "
              "complex concepts into clear, engaging articles.",
    verbose=True
)

# 定义 Task —— 串行依赖
research_task = Task(
    description="Research the latest AI Agent developments in 2026, "
                "focusing on AutoGen, CrewAI, and LangGraph.",
    expected_output="A bullet list of 5 key trends with explanations.",
    agent=researcher
)

write_task = Task(
    description="Write a 500-word article based on the research findings.",
    expected_output="A polished 500-word article in markdown format.",
    agent=writer,
    context=[research_task]  # 依赖调研结果
)

# 组建 Crew 并启动
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process=Process.sequential,
    verbose=True
)

result = crew.kickoff()
print(result)

模式二：自定义 Tool 开发

CrewAI 支持通过继承 BaseTool 类开发自定义工具，让 Agent 具备特定的能力。下面是一个完整的自定义工具示例，展示如何创建一个数据库查询工具和情感分析工具，并挂载到 Agent 上。自定义工具让 CrewAI 的 Agent 能够对接任何外部系统，极大扩展了框架的适用范围：

Python: CrewAI 自定义 Tool 
from crewai import Agent, Task, Crew
from crewai.tools import BaseTool

# 自定义工具：数据库查询
class DatabaseQueryTool(BaseTool):
    name: str = "Database Query Tool"
    description: str = "Query the product database for sales data."

    def _run(self, query: str) -> str:
        # 模拟数据库查询
        if "sales" in query.lower():
            return "Q1: 120万, Q2: 150万, Q3: 180万, Q4: 210万"
        elif "inventory" in query.lower():
            return "库存: iPhone 500台, iPad 300台, Mac 200台"
        else:
            return f"Query result for: {query}"

# 自定义工具：情感分析
class SentimentAnalysisTool(BaseTool):
    name: str = "Sentiment Analysis Tool"
    description: str = "Analyze sentiment of customer reviews."

    def _run(self, text: str) -> str:
        positive_words = ["好", "棒", "优秀", "满意"]
        score = sum(1 for w in positive_words if w in text)
        sentiment = "正面" if score > 0 else "中性"
        return f"情感分析结果: {sentiment} (正面词数: {score})"

# 创建挂载自定义工具的 Agent
analyst = Agent(
    role="Data Analyst",
    goal="Analyze sales data and customer sentiment",
    backstory="You are a data analyst with expertise in sales trends.",
    tools=[DatabaseQueryTool(), SentimentAnalysisTool()],
    verbose=True
)

# 创建任务并执行
analysis_task = Task(
    description="Query sales data and analyze customer sentiment.",
    expected_output="A summary report with sales trends and sentiment.",
    agent=analyst
)

crew = Crew(agents=[analyst], tasks=[analysis_task], verbose=True)
result = crew.kickoff()

模式三：层级执行（Hierarchical Process）

当任务流程不是简单线性时，CrewAI 的 hierarchical 模式引入了一个 manager Agent 来动态调度任务。Manager 会根据任务内容和 Agent 能力自动分配，适合多分支调研和交叉审核场景。这种模式更接近真实团队中项目经理的调度方式，比 sequential 模式更灵活：

Python: CrewAI 层级执行模式 
from crewai import Agent, Task, Crew, Process

# 定义多个专业 Agent
researcher = Agent(
    role="Market Researcher",
    goal="Gather market data and competitor info",
    backstory="10 years market research experience.",
    verbose=True
)

analyst = Agent(
    role="Financial Analyst",
    goal="Analyze financial viability and ROI",
    backstory="CPA with expertise in investment analysis.",
    verbose=True
)

strategist = Agent(
    role="Strategy Advisor",
    goal="Synthesize findings into actionable strategy",
    backstory="Former McKinsey consultant specializing in tech.",
    verbose=True
)

# 定义任务（不指定固定顺序，由 manager 调度）
tasks = [
    Task(
        description="Research the AI Agent market size and competitors.",
        expected_output="Market analysis with 3 key competitors.",
        agent=researcher
    ),
    Task(
        description="Analyze ROI for entering the AI Agent market.",
        expected_output="Financial viability report.",
        agent=analyst
    ),
    Task(
        description="Synthesize all findings into a go/no-go strategy.",
        expected_output="Strategic recommendation document.",
        agent=strategist
    )
]

# 使用 hierarchical 模式 —— manager 自动调度
crew = Crew(
    agents=[researcher, analyst, strategist],
    tasks=tasks,
    process=Process.hierarchical,
    verbose=True
)

result = crew.kickoff()

📋 八股总结 — 面试高频考点
Q1: AutoGen 和 CrewAI 的设计哲学有什么不同？
AutoGen：对话驱动。Agent 之间通过群聊对话协作，自由决定谁说话、说什么。像头脑风暴会议，灵活但不太可控。

CrewAI：角色驱动。每个 Agent 有明确角色（Role）、目标（Goal）和背景（Backstory），任务按流程分配。像项目团队分工，可控但不够灵活。

选择：探索性任务选 AutoGen，标准化流程选 CrewAI。
Q2: AutoGen 的 GroupChatManager 作用是什么？
GroupChatManager 是群聊的"主持人"，管理多个 Agent 的对话：
① 决定下一个发言的 Agent（可基于 LLM 判断或轮转）
② 控制对话轮次（max_round 防止无限对话）
③ 判断终止条件（任务完成或达到轮次上限）

没有它，Agent 之间的对话会混乱无序。
Q3: CrewAI 中 Agent 的 Role、Goal、Backstory 各自的作用？
Role：Agent 的身份标签（如"资深研究员"），影响 LLM 的行为风格。
Goal：Agent 的核心目标（如"找到最准确的信息"），指导决策方向。
Backstory：背景故事（如"10年研究经验"），让 LLM 进入角色，影响输出风格和专业度。

三者组合就像给 LLM 一份"角色剧本"，让它更专业、更聚焦。
Q4: AutoGen 的 UserProxyAgent 有什么特殊作用？
UserProxyAgent 是人类和 Agent 系统之间的"桥梁"：
① 代码执行：可以实际运行 Python 代码并返回结果
② 人类代理：human_input_mode 设为 "ALWAYS" 时，每步都向人类确认
③ 工具调用：可以执行函数调用

它是 AutoGen 中唯一能"动手"的 Agent，其他 AssistantAgent 只能"说话"。
Q5: AutoGen、CrewAI、LangGraph 三个框架怎么选？
LangGraph：要精确控制流程、需要人工干预、需要可视化调试 → 生产级 Agent
AutoGen：需要 Agent 自由对话、需要代码执行、探索性任务 → 研究和原型
CrewAI：需要角色化分工、标准化流程、快速上手 → 内容生产、自动化工作流

没有最好的框架，只有最合适的场景。
Q6: AutoGen 的 GroupChat 模式如何工作？
GroupChat 模式通过 GroupChatManager 统一管理多个 Agent 的对话流程：
① 注册阶段：将所有 Agent 加入 GroupChat 的 agents 列表，并创建 GroupChatManager 作为管理者
② 发言调度：Manager 根据对话上下文和 Agent 的 system_message，决定下一个发言的 Agent（也可设置轮转模式）
③ 消息广播：当前发言 Agent 的消息会广播给所有其他 Agent，确保信息共享
④ 轮次控制：通过 max_round 参数限制最大对话轮次，防止无限循环
⑤ 终止判断：当某个 Agent 发出包含终止关键词的消息，或达到最大轮次时，对话结束

本质上就是模拟人类群聊：有个主持人决定谁说话，每个人都能听到所有人的发言。
Q7: CrewAI 的 Process.sequential 和 Process.hierarchical 区别？
Process.sequential（顺序执行）：
- 任务按 tasks 列表的顺序依次执行，前一个任务完成后才执行下一个
- 前序任务的输出自动作为后序任务的上下文（配合 context 参数）
- 不需要额外的管理 Agent，流程简单直接
- 适合线性工作流：调研 → 写作 → 审核

Process.hierarchical（层级执行）：
- 引入一个 manager Agent 作为管理者，负责分配任务给最合适的 Agent
- Manager 会根据任务内容和 Agent 能力动态调度，不按固定顺序
- 支持更复杂的任务依赖关系和并行可能性
- 适合非线性工作流：多分支调研、交叉审核

选择建议：流程固定用 sequential，需要动态调度用 hierarchical。
Q8: AutoGen 如何实现 Human-in-the-loop？human_input_mode 有哪些值？
AutoGen 通过 UserProxyAgent 的 human_input_mode 参数实现 Human-in-the-loop：

"ALWAYS"：每轮对话后都暂停，等待人类输入。适合需要严格人工审核的场景，如高风险决策。
"TERMINATE"（默认）：仅在对话即将终止时暂停，让人类确认是否结束。平衡了自动化和可控性。
"NEVER"：完全不等待人类输入，全自动运行。适合无人值守的自动化任务。

此外，is_termination_msg 参数可以自定义终止判断函数，比如检测消息中是否包含特定关键词。结合 human_input_mode 可以灵活控制人工介入的粒度——从全自动到每步确认，满足不同场景需求。
Q9: CrewAI 的 Agent 必须配置哪些字段？
CrewAI 的 Agent 必须配置以下三个核心字段：
① role：Agent 的角色身份（如"Senior Research Analyst"），决定了 LLM 的行为风格和专业方向
② goal：Agent 的核心目标（如"Uncover cutting-edge developments"），指导 Agent 的决策方向
③ backstory：背景故事（如"10 years experience analyst"），让 LLM 进入角色，影响输出的专业度和风格

可选字段包括：tools（工具列表）、verbose（日志开关）、allow_delegation（是否允许委托）。这三个必填字段缺一不可，共同构成 Agent 的"角色剧本"。
Q10: AutoGen vs CrewAI 怎么选？
从四个维度判断：
① 需要代码执行 → AutoGen（UserProxyAgent 原生支持代码执行，CrewAI 需要通过 Tool 间接实现）
② 需要标准化流程 → CrewAI（sequential/hierarchical 模式提供强流程控制，AutoGen 的对话模式不够可控）
③ 需要人工介入 → AutoGen（human_input_mode 提供三个级别的人工介入，CrewAI 支持有限）
④ 需要角色化分工 → CrewAI（role/goal/backstory 三件套让角色定义清晰，AutoGen 的 system_message 粒度较粗）

实际项目中可以两者结合：用 AutoGen 做探索性原型，用 CrewAI 搭建标准化生产流程。
