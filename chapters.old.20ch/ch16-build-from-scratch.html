16.6 意图识别增强

Agent 的第一步是理解用户想做什么。意图识别的质量决定了后续工具选择、上下文填充、输出格式的准确性。没有好的意图识别，Agent 就像一个"听不懂话"的助手——即便能力很强，也无法正确发挥。

意图分类体系

🎯 三类意图及其特征
类型 定义 示例 处理策略
显式意图 用户直接表达了目标 "帮我查北京天气"、"翻译这段话" 直接映射到工具，无需推断
隐式意图 用户描述了场景而非直接需求 "我明天要去上海出差"、"最近股市不太行" 需推断→查天气/订票/看行情
复合意图 一句话包含多个子目标 "查天气+订机票+推荐酒店" 拆解为子意图序列，按依赖排序

意图消歧策略

当一句话可能对应多个意图时，需要消歧——选择最合理的解释：

💡 四种消歧方法

① 上下文推断：利用对话历史判断。"帮我订"→上文提到出差→订机票而非订外卖。
② 优先级排序：预设意图优先级。歧义时选最高优先级意图，低优先级作为备选。
③ 主动确认：不确定时反问用户。"您是要查天气还是订机票？"——简单但最可靠。
④ 多意图并行：无法消歧时同时执行多个候选，合并结果让用户选择。

多意图识别与拆解

复合意图需要拆解为有序子意图序列。拆解的核心是依赖分析：哪些子意图可以并行执行，哪些必须串行等待前置结果。

import re
from enum import Enum
from typing import List, Optional

class IntentType(Enum):
    EXPLICIT = "explicit"    # 显式意图
    IMPLICIT = "implicit"    # 隐式意图
    COMPOSITE = "composite"  # 复合意图

class Intent:
    def __init__(self, name: str, type: IntentType,
                 tool: str, params: dict,
                 depends_on: List[str] = None):
        self.name = name
        self.type = type
        self.tool = tool        # 映射的工具名
        self.params = params    # 提取的参数
        self.depends_on = depends_on or []  # 依赖的前置意图

class IntentClassifier:
    """意图分类器：识别、消歧、拆解"""

    # 意图模式库（可扩展为 LLM 分类）
    INTENT_PATTERNS = {
        "check_weather": {
            "patterns": [r"天气", r"气温", r"下雨", r"穿什么"],
            "tool": "weather_api",
            "param_keys": ["city", "date"],
            "priority": 1
        },
        "book_flight": {
            "patterns": [r"订机票", r"航班", r"飞去", r"出差"],
            "tool": "flight_booking",
            "param_keys": ["origin", "destination", "date"],
            "priority": 2
        },
        "search_hotel": {
            "patterns": [r"酒店", r"住宿", r"住哪", r"民宿"],
            "tool": "hotel_search",
            "param_keys": ["city", "budget", "date"],
            "priority": 3
        },
        "translate": {
            "patterns": [r"翻译", r"translate", r"英文怎么说"],
            "tool": "translate_api",
            "param_keys": ["text", "target_lang"],
            "priority": 4
        }
    }

    def classify(self, user_input: str,
                 context: List[str] = None) -> List[Intent]:
        """分类用户输入，返回意图列表"""
        matched = []

        for intent_name, config in self.INTENT_PATTERNS.items():
            for pattern in config["patterns"]:
                if re.search(pattern, user_input):
                    params = self._extract_params(
                        user_input, config["param_keys"]
                    )
                    matched.append(Intent(
                        name=intent_name,
                        type=self._determine_type(user_input, matched),
                        tool=config["tool"],
                        params=params
                    ))
                    break  # 每个意图只匹配一次

        # 消歧：多个候选时选最高优先级
        if len(matched) > 1:
            matched = self._disambiguate(matched, context)

        # 复合意图拆解：建立依赖关系
        if len(matched) > 1:
            matched = self._resolve_dependencies(matched)

        # 无匹配 → 主动确认
        if not matched:
            return [Intent(
                name="clarify",
                type=IntentType.EXPLICIT,
                tool="ask_user",
                params={"question": "您是想查天气、订票，还是其他？"}
            )]

        return matched

    def _determine_type(self, input: str,
                        existing: List[Intent]) -> IntentType:
        """判断意图类型"""
        if len(existing) >= 1:
            return IntentType.COMPOSITE  # 多意图=复合
        direct_keywords = ["查", "订", "翻译", "帮我"]
        if any(kw in input for kw in direct_keywords):
            return IntentType.EXPLICIT
        return IntentType.IMPLICIT

    def _disambiguate(self, intents: List[Intent],
                      context: List[str]) -> List[Intent]:
        """消歧：利用上下文和优先级"""
        if context:
            # 上下文推断：如果上文提到出差→优先订机票
            for msg in context[-3:]:  # 最近3轮
                if "出差" in msg:
                    business = [i for i in intents
                                if i.name == "book_flight"]
                    if business:
                        return business  # 只保留订机票

        # 无上下文 → 按优先级排序，保留top-3
        sorted_intents = sorted(
            intents,
            key=lambda i: self.INTENT_PATTERNS[i.name]["priority"]
        )
        return sorted_intents[:3]

    def _resolve_dependencies(self,
                              intents: List[Intent]) -> List[Intent]:
        """建立意图间的依赖关系"""
        # 简单规则：订机票依赖查天气结果
        for intent in intents:
            if intent.name == "search_hotel":
                intent.depends_on = ["book_flight"]
            if intent.name == "check_transport":
                intent.depends_on = ["search_hotel"]

        return intents

    def _extract_params(self, input: str,
                        param_keys: List[str]) -> dict:
        """从输入中提取参数（简化版）"""
        params = {}
        # 城市提取
        cities = ["上海", "北京", "广州", "深圳", "杭州"]
        for city in cities:
            if city in input:
                params["city"] = city
                break
        # 日期提取
        date_patterns = [r"明天", r"后天", r"下周", r"今晚"]
        for pat in date_patterns:
            if re.search(pat, input):
                params["date"] = re.search(pat, input).group()
                break
        return params


# ===== 使用示例 =====
classifier = IntentClassifier()

# 显式意图

# → [Intent(name="check_weather", type=EXPLICIT, tool="weather_api")]

# 隐式意图

# → [Intent(name="book_flight", type=IMPLICIT, ...), 
#    Intent(name="check_weather", type=COMPOSITE, ...)]

# 复合意图（带上下文消歧）
result = classifier.classify(

    context=["我下周要去上海出差"]
)
# → [Intent(name="book_flight", depends_on=[]),
#    Intent(name="check_weather", depends_on=[]),
#    Intent(name="search_hotel", depends_on=["book_flight"])]

16.7 上下文工程实战

Context Engineering 是 Agent 性能的关键杠杆。LLM 的输出质量与输入的上下文质量直接相关——垃圾上下文产出垃圾结果，精心组织的上下文产出精准结果。上下文工程的核心是在有限的 Token 窗口内，放入最相关、最结构化的信息。

🔑 核心原则：上下文不是越多越好，而是越精准越好。100 tokens 精准上下文 > 10000 tokens 无关废话。LLM 的注意力机制会稀释在噪声中——无关信息不仅浪费 Token，还会降低输出质量。

上下文压缩器实现 
from dataclasses import dataclass, field
from typing import List, Dict, Optional
import json

@dataclass
class Message:
    role: str       # "user" | "assistant" | "system"
    content: str
    metadata: Dict = field(default_factory=dict)

class ContextCompressor:
    """上下文压缩器：智能压缩对话历史"""

    def __init__(self, max_tokens: int = 4000,
                 summary_model: str = "gpt-4o-mini"):
        self.max_tokens = max_tokens
        self.summary_model = summary_model
        self.key_info_tags = ["决策", "确认", "偏好", "重要", "结论"]

    def compress(self, messages: List[Message],
                 current_query: str) -> List[Message]:
        """压缩对话历史，适配当前查询"""
        # 第1步：计算当前 Token 量
        total_tokens = self._estimate_tokens(messages)

        if total_tokens <= self.max_tokens:
            return messages  # 不需要压缩

        # 第2步：分离近期对话（保留）和远期对话（压缩）
        recent = messages[-6:]  # 最近6轮原样保留
        older = messages[:-6]   # 更早的压缩

        # 第3步：对远期对话做结构化摘要
        summary = self._summarize_older(older, current_query)

        # 第4步：选择性保留关键片段
        key_fragments = self._extract_key_fragments(older)

        # 第5步：组装压缩后的上下文
        compressed = [
            Message(role="system", content=summary),
            Message(role="system",
                    content=f"关键信息摘要：\n{key_fragments}"),
            *recent  # 近期对话原样保留
        ]

        return compressed

    def _summarize_older(self, older: List[Message],
                         current_query: str) -> str:
        """将旧对话压缩为结构化摘要"""
        # 按主题分段摘要
        segments = self._segment_by_topic(older)

        summaries = []
        for topic, msgs in segments.items():
            # 只摘要与当前查询相关的主题
            if self._is_relevant(topic, current_query):
                summary_text = self._llm_summarize(msgs)
                summaries.append(f"【{topic}】{summary_text}")

        if not summaries:
            # 无相关主题 → 通用摘要
            summaries.append(
                self._llm_summarize(older, focus="关键决策和结论")
            )

        return "对话历史摘要：\n" + "\n".join(summaries)

    def _extract_key_fragments(self,
                               older: List[Message]) -> str:
        """提取带关键标记的消息片段"""
        key_items = []

        for msg in older:
            # 提取包含关键标记的内容
            for tag in self.key_info_tags:
                if tag in msg.content:
                    # 只保留关键句，丢弃上下文废话
                    key_sentence = self._extract_key_sentence(
                        msg.content, tag
                    )
                    key_items.append(f"- [{tag}] {key_sentence}")

        return "\n".join(key_items) if key_items else "无特别记录"

    def _segment_by_topic(self,
                          messages: List[Message]) -> Dict:
        """按主题分段"""
        segments = {}
        current_topic = "通用"

        for msg in messages:
            # 简单主题检测（可用LLM增强）
            topic_keywords = {
                "天气": "天气查询",
                "机票": "机票预订",
                "酒店": "酒店推荐",
                "翻译": "翻译服务",
                "技术": "技术调研"
            }
            detected = None
            for kw, topic in topic_keywords.items():
                if kw in msg.content:
                    detected = topic
                    break

            topic = detected or current_topic
            if topic not in segments:
                segments[topic] = []
            segments[topic].append(msg)

        return segments

    def _is_relevant(self, topic: str,
                     current_query: str) -> bool:
        """判断旧主题是否与当前查询相关"""
        # 简单相关性检测
        topic_words = topic.split()
        query_words = current_query.split()
        overlap = set(topic_words) & set(query_words)
        return len(overlap) > 0

    def _estimate_tokens(self,
                         messages: List[Message]) -> int:
        """估算 Token 数（中文≈1.5字/token）"""
        total_chars = sum(len(m.content) for m in messages)
        return int(total_chars * 1.5)

    def _llm_summarize(self, messages: List[Message],
                       focus: str = "要点") -> str:
        """调用LLM生成摘要"""
        text = "\n".join(
            f"{m.role}: {m.content}" for m in messages
        )
        prompt = f"将以下对话压缩为3-5句话的摘要，\n"
        prompt += f"重点保留{focus}：\n{text}"
        # 实际实现中调用LLM
        # return llm.invoke(prompt)
        return f"[摘要] 关于{focus}的关键结论..."

    def _extract_key_sentence(self, content: str,
                              tag: str) -> str:
        """从长文本中提取包含标记的关键句"""
        sentences = content.split("。")
        for s in sentences:
            if tag in s:\n                return s.strip() + "。"
        return content[:50]  # 无匹配时取前50字

16.8 消息压缩与记忆治理

Agent 长时间运行后，对话历史不断膨胀——这就是Context Rot 问题：上下文腐烂，信息过期、重复、噪声堆积，导致 Agent 性能持续退化。记忆治理就是解决这个问题。

🦠 Context Rot 的三大症状

① 信息过期：旧决策不再有效（"用Python"→后来改用Java了，但Agent还记着Python）
② 信息重复：同一件事反复确认（每轮都问"你是要Python吗？"）
③ 噪声堆积：寒暄、错误尝试、无关讨论淹没关键信息（20轮对话中只有3轮有用）

四步压缩流程

🔧 四步压缩法：锁定→完整→连贯→提示
步骤 动作 目的 示例
第1步：锁定核心 识别不可丢弃的信息 防止压缩丢掉关键决策 用户偏好、已确认结论、待解决问题
第2步：确保完整性 摘要覆盖所有关键主题 压缩后不丢失任何重要话题 每段对话至少一句话摘要
第3步：保持连贯性 摘要按时间线组织 Agent理解因果链条 "先调研→再对比→最终决策：选LangGraph"
第4步：添加系统提示 告知Agent这是压缩后的历史 Agent正确使用压缩上下文 "以下是压缩摘要，包含核心决策和结论"

from collections import deque
from typing import List, Optional

class SlidingWindowMemory:
    """滑动窗口记忆管理器"""

    def __init__(self, window_size: int = 10,
                 compress_threshold: int = 8):
        self.window_size = window_size       # 保留最近N轮
        self.compress_threshold = compress_threshold  # 触发压缩的轮数
        self.recent_window = deque(maxlen=window_size)
        self.compressed_summary: Optional[str] = None
        self.key_decisions: List[str] = []    # 不可丢弃的决策
        self.user_preferences: Dict[str, str] = {}  # 用户偏好

    def add_message(self, role: str, content: str):
        """添加新消息"""
        # 先检查是否包含关键决策
        self._extract_decision(content)
        # 再检查是否包含用户偏好
        self._extract_preference(content)
        # 加入滑动窗口
        self.recent_window.append((role, content))

        # 超过阈值 → 触发压缩
        if len(self.recent_window) >= self.compress_threshold:
            self._compress_older()

    def get_context(self, current_query: str) -> str:
        """获取当前对话上下文"""
        context_parts = []

        # 第1步：系统提示（告知压缩状态）
        context_parts.append(
            "[系统] 以下是压缩后的对话历史，"
            "包含核心决策和最近对话。"
        )

        # 第2步：压缩摘要（确保完整性）
        if self.compressed_summary:
            context_parts.append(
                f"[历史摘要]\n{self.compressed_summary}"
            )

        # 第3步：关键决策（锁定核心）
        if self.key_decisions:
            decisions = "\n".join(
                f"- {d}" for d in self.key_decisions
            )
            context_parts.append(
                f"[关键决策]\n{decisions}"
            )

        # 第4步：用户偏好（锁定核心）
        if self.user_preferences:
            prefs = "\n".join(
                f"- {k}: {v}"
                for k, v in self.user_preferences.items()
            )
            context_parts.append(
                f"[用户偏好]\n{prefs}"
            )

        # 第5步：最近对话（保持连贯性）
        recent = "\n".join(
            f"{role}: {content}"
            for role, content in self.recent_window
        )
        context_parts.append(f"[最近对话]\n{recent}")

        return "\n\n".join(context_parts)

    def _compress_older(self):
        """压缩超出窗口的旧对话"""
        # 保留最近N轮，压缩更早的
        older = list(self.recent_window)[:-self.window_size]

        if not older:
            return

        # 生成结构化摘要
        new_summary = self._generate_summary(older)

        # 合并到已有摘要（保持连贯性）
        if self.compressed_summary:
            self.compressed_summary = (
                f"{self.compressed_summary}\n\n"
                f"后续进展：\n{new_summary}"
            )
        else:
            self.compressed_summary = new_summary

        # 清理窗口中已压缩的消息
        while len(self.recent_window) > self.window_size:
            self.recent_window.popleft()

    def _generate_summary(self, older: List) -> str:
        """对旧对话生成结构化摘要"""
        topics = {}
        for role, content in older:
            # 检测主题（简化版，实际可用LLM）
            topic = self._detect_topic(content)
            if topic not in topics:
                topics[topic] = []
            topics[topic].append(f"{role}: {content}")

        summary_lines = []
        for topic, msgs in topics.items():
            # 每个主题1-2句摘要
            summary_lines.append(
                f"关于{topic}：讨论了{len(msgs)}轮，"
                f"结论是{self._extract_conclusion(msgs)}"
            )

        return "\n".join(summary_lines)

    def _extract_decision(self, content: str):
        """提取关键决策"""
        decision_markers = [
            "决定", "确认", "选择", "放弃",
            "最终方案", "拍板"
        ]
        for marker in decision_markers:
            if marker in content:
                self.key_decisions.append(content[:80])
                break

    def _extract_preference(self, content: str):
        """提取用户偏好"""
        pref_patterns = {
            "Python": "技术栈=Python",
            "Java": "技术栈=Java",
            "生产": "场景=生产环境",
            "研究": "场景=研究探索"
        }
        for kw, pref in pref_patterns.items():
            if kw in content:
                key, value = pref.split("=")
                self.user_preferences[key] = value

    def _detect_topic(self, content: str) -> str:
        """简单主题检测"""
        topic_map = {
            "天气": "天气查询",
            "机票": "机票预订",
            "LangGraph": "技术调研",
            "AutoGen": "技术调研",
        }
        for kw, topic in topic_map.items():
            if kw in content:
                return topic
        return "通用"

    def _extract_conclusion(self, msgs: List[str]) -> str:
        """提取对话结论"""
        # 查找最后一条assistant消息中的结论性内容
        for msg in reversed(msgs):
            if msg.startswith("assistant:"):
                return msg.split(":")[-1][:50]
        return "待确认"

向量库归档实现 
from datetime import datetime
from typing import List, Dict

class VectorArchive:
    """向量库归档：长期记忆存储与召回"""

    def __init__(self, vector_store, top_k: int = 5):
        self.vector_store = vector_store  # 如 Chroma/Pinecone
        self.top_k = top_k

    def archive(self, messages: List[Dict],
                session_id: str) -> str:
        """归档对话到向量库"""
        # 按主题分段，每段独立存储
        segments = self._segment(messages)

        doc_ids = []
        for topic, content in segments.items():
            # 生成元数据（方便精准检索）
            metadata = {
                "session_id": session_id,
                "topic": topic,
                "timestamp": datetime.now().isoformat(),
                "type": "conversation_archive"
            }

            # 存入向量库
            doc_id = self.vector_store.add_document(
                text=content,
                metadata=metadata
            )
            doc_ids.append(doc_id)

        return f"归档完成：{len(doc_ids)}个段落"

    def recall(self, query: str,
               context_filter: Dict = None) -> List[str]:
        """从向量库召回相关信息"""
        # 构建检索条件
        search_filter = {}
        if context_filter:
            search_filter.update(context_filter)

        # 检索最相关的top_k段落
        results = self.vector_store.similarity_search(
            query=query,
            k=self.top_k,
            filter=search_filter
        )

        # 按相关度排序返回
        return [
            f"[{r.metadata['topic']}] {r.text}"
            for r in results
        ]

    def _segment(self, messages: List[Dict]) -> Dict:
        """按主题分段存储"""
        segments = {}
        current_topic = "general"

        for msg in messages:
            topic = self._detect_topic(msg["content"])
            if topic != current_topic:
                current_topic = topic
            if current_topic not in segments:
                segments[current_topic] = ""
            segments[current_topic] += (
                f"\n{msg['role']}: {msg['content']}"
            )

        return segments

    def _detect_topic(self, content: str) -> str:
        """检测主题标签"""
        topic_map = {
            "天气": "weather",
            "机票": "flight",
            "酒店": "hotel",
            "LangGraph": "langgraph",
            "AutoGen": "autogen"
        }
        for kw, topic in topic_map.items():
            if kw in content:
                return topic
        return "general"

    """工具参数模型"""
    city: Optional[str] = Field(
        None, description="城市名"
    )

验证失败重试策略

🔄 三层重试策略
第1层：自动重试

验证失败时，将错误信息反馈给LLM，要求修正输出。
最多3次，每次附带验证错误详情。

第2层：降级兜底

重试失败后，用正则/关键词提取兜底。
从自由文本中尽力提取关键字段。

第3层：人工确认

兜底也失败时，暂停并请求用户确认。
展示原始输出，让用户手动选择。

三层重试实现 
class StructuredOutputProcessor:
    """结构化输出处理器：验证+重试"""

    def __init__(self, model, schema: type,
                 max_retries: int = 3):
        self.model = model
        self.schema = schema  # Pydantic模型类
        self.max_retries = max_retries

    def process(self, prompt: str) -> dict:
        """处理LLM输出，确保结构化"""
        # 第1层：尝试LLM结构化输出
        for attempt in range(self.max_retries):
            raw = self._call_llm(prompt, attempt)
            try:
                result = self.schema(**raw)
                return result.dict()  # 验证成功
            except ValidationError as e:\n                # 验证失败 → 反馈错误，重试\n                prompt = self._add_error_feedback(\n                    prompt, raw, str(e)\n                )\n                print(f"重试 {attempt+1}/{self.max_retries}")

        # 第2层：降级兜底（正则提取）
        fallback_result = self._fallback_extract(raw)
        if fallback_result:
            return fallback_result

        # 第3层：人工确认
        return self._request_human_confirmation(raw)

    def _call_llm(self, prompt: str,
                  attempt: int) -> dict:
        """调用LLM获取结构化输出"""
        system_msg = (
            "请按JSON格式输出，严格遵守以下Schema：\n"
            f"{json.dumps(self.schema.schema(), indent=2)}"
        )
        if attempt > 0:
            system_msg += (
                "\n注意：上次输出验证失败，请修正错误。"
            )
        response = self.model.invoke(
            system_msg + "\n" + prompt
        )
        return json.loads(response)

    def _add_error_feedback(self, prompt: str,
                            raw: dict, error: str) -> str:
        """将验证错误反馈给LLM"""
        return (
            f"{prompt}\n\n"
            f"[上次输出] {json.dumps(raw)}\n"
            f"[验证错误] {error}\n"
            f"请修正上述错误，重新输出。"
        )

    def _fallback_extract(self, raw_text: str) -> dict:
        """降级兜底：正则+关键词提取"""
        # 尝试从自由文本中提取关键字段
        import re
        result = {}

        # 城市提取
        city_match = re.search(
            r"城市[是为：](\w+)", raw_text
        )
        if city_match:
            result["city"] = city_match.group(1)

        # 日期提取
        date_match = re.search(
            r"(明天|后天|下周|今晚)", raw_text
        )
        if date_match:
            result["date"] = date_match.group(1)

        # 意图提取
        intent_keywords = {
            "天气": "check_weather",
            "机票": "book_flight",
            "酒店": "search_hotel",
            "翻译": "translate"
        }
        for kw, intent in intent_keywords.items():
            if kw in raw_text:
                result["intent"] = intent
                break

        return result if len(result) >= 2 else None

    def _request_human_confirmation(self,
                                    raw: dict) -> dict:
        """人工确认兜底"""
        return {
            "intent": "clarify",
            "tool": "ask_user",
            "params": {
                "question": (
                    f"无法自动识别，原始输出：{raw}\n"
                    "请确认您的意图。"
                )
            },
            "confidence": 0.0,
            "reasoning": "自动验证失败，需人工确认"
        }

from typing import TypedDict, Annotated, Optional
from enum import Enum
from pydantic import BaseModel, Field
import operator, json

# ===== 全流程组件定义 =====

# 1. 意图识别模块（16.6）
class IntentEnum(str, Enum):
    CHECK_WEATHER = "check_weather"
    BOOK_FLIGHT = "book_flight"
    SEARCH_HOTEL = "search_hotel"
    TRANSLATE = "translate"
    CLARIFY = "clarify"
    UNKNOWN = "unknown"

class IntentResult(BaseModel):
    intent: IntentEnum
    tool: str
    params: dict = Field(default_factory=dict)
    confidence: float = Field(ge=0, le=1)
    reasoning: str = ""
    depends_on: list = Field(default_factory=list)

# 2. Agent State（整合所有模块的状态）
class AgentState(TypedDict):
    user_input: str                           # 原始输入
    intents: list                             # 识别的意图列表
    context: str                              # 压缩后的上下文
    tool_calls: list                          # 工具调用列表
    tool_results: list                        # 工具执行结果
    structured_output: dict                   # 结构化输出
    memory_update: dict                       # 记忆更新指令
    messages: Annotated[list, operator.add]   # 对话历史

# 3. 工具定义
INTENT_TOOL_MAP = {
    "check_weather": {
        "primary": "weather_api",
        "fallback": "web_search",
        "param_keys": ["city", "date"]
    },
    "book_flight": {
        "primary": "flight_booking",
        "fallback": "web_search",
        "param_keys": ["origin", "destination", "date"]
    },
    "search_hotel": {
        "primary": "hotel_search",
        "fallback": "review_aggregator",
        "param_keys": ["city", "budget", "date"]
    },
    "translate": {
        "primary": "translate_api",
        "fallback": "llm_direct",
        "param_keys": ["text", "target_lang"]
    }
}

@tool
def weather_api(city: str, date: str) -> str:
    """查天气API"""
    return f"{city}{date}天气：晴，25°C"

@tool
def flight_booking(origin: str, destination: str,
                   date: str) -> str:
    """订机票API"""
    return f"{origin}→{destination} {date}航班已预订"

@tool
def hotel_search(city: str, budget: str,
                 date: str) -> str:
    """搜酒店API"""
    return f"{city}推荐酒店：xxx酒店，¥{budget}/晚"

@tool
def web_search(query: str) -> str:
    """网络搜索（万能兜底）"""
    return f"搜索结果：{query}的相关信息..."

@tool
def ask_user(question: str) -> str:
    """主动确认"""
    return question

# ===== LangGraph 工作流定义 =====

def intent_recognition_node(state: AgentState):
    """节点1：意图识别（16.6）"""
    classifier = IntentClassifier()
    intents = classifier.classify(
        state["user_input"],
        context=[m.content for m in state["messages"][-6:]]
    )
    # 转为结构化 IntentResult
    intent_results = [
        IntentResult(
            intent=i.name,
            tool=i.tool,
            params=i.params,
            confidence=0.85,
            reasoning=f"从输入中识别到{i.type.value}意图"
        ).dict()
        for i in intents
    ]
    return {"intents": intent_results}

def context_management_node(state: AgentState):
    """节点2：上下文管理（16.7-16.8）"""
    compressor = ContextCompressor(max_tokens=4000)
    memory = SlidingWindowMemory(window_size=10)

    # 加入当前对话历史
    for msg in state["messages"]:
        memory.add_message(msg.role, msg.content)

    # 获取压缩上下文
    context = memory.get_context(state["user_input"])
    return {"context": context}

def tool_selection_node(state: AgentState):
    """节点3：工具选择"""
    tool_calls = []

    for intent in state["intents"]:
        intent_name = intent["intent"]
        if intent_name in INTENT_TOOL_MAP:
            config = INTENT_TOOL_MAP[intent_name]
            tool_calls.append({
                "tool": config["primary"],
                "params": intent["params"],
                "fallback": config["fallback"],
                "confidence": intent["confidence"]
            })
        else:
            # 未知意图 → 主动确认
            tool_calls.append({
                "tool": "ask_user",
                "params": {"question": "您是想做什么？"},
                "fallback": None,
                "confidence": 0.0
            })

    return {"tool_calls": tool_calls}

def tool_execution_node(state: AgentState):
    """节点4：工具执行+验证（16.9）"""
    results = []

    for call in state["tool_calls"]:
        # 执行主工具
        try:
            tool_func = TOOL_REGISTRY[call["tool"]]
            result = tool_func(**call["params"])
            results.append({
                "tool": call["tool"],
                "params": call["params"],
                "result": result,
                "status": "success"
            })
        except Exception as e:\n            # 主工具失败 → 用备选工具\n            if call.get("fallback"):
                fallback_func = TOOL_REGISTRY[call["fallback"]]
                fallback_params = {
                    "query": str(call["params"])
                }
                result = fallback_func(**fallback_params)
                results.append({
                    "tool": call["fallback"],
                    "params": fallback_params,
                    "result": result,
                    "status": "fallback_success"
                })
            else:
                # 无备选 → 记录失败
                results.append({
                    "tool": call["tool"],
                    "params": call["params"],
                    "result": str(e),
                    "status": "failed"
                })

    # 结构化输出验证
    processor = StructuredOutputProcessor(
        model=llm, schema=IntentResult, max_retries=3
    )
    validated = []
    for r in results:
        try:
            validated_result = IntentResult(**r)
            validated.append(validated_result.dict())
        except ValidationError:
            # 验证失败 → 三层重试
            fixed = processor.process(str(r))
            validated.append(fixed)

    return {"tool_results": results,
            "structured_output": validated}

def memory_governance_node(state: AgentState):
    """节点5：记忆治理（16.8）"""
    # 更新STATE.md
    state_updates = {
        "current_task": state["user_input"],
        "completed_intents": [
            i["intent"] for i in state["intents"]
        ],
        "pending_intents": [
            i["intent"] for i in state["intents"]
            if i.get("depends_on")
        ],
        "key_decisions": []
    }

    # 提取决策
    for result in state["tool_results"]:
        if result["status"] == "success":
            state_updates["key_decisions"].append(
                f"{result['tool']}: {result['result'][:50]}"
            )

    # 归档到向量库（超长对话）
    archive = VectorArchive(vector_store=chroma)
    if len(state["messages"]) > 20:
        archive.archive(
            state["messages"][:10],
            session_id=state.get("session_id", "default")
        )

    # 压缩对话历史
    compressor = ContextCompressor(max_tokens=4000)
    compressed = compressor.compress(
        state["messages"],
        state["user_input"]
    )

    return {
        "memory_update": state_updates,
        "messages": compressed
    }

def response_generation_node(state: AgentState):
    """节点6：生成最终响应"""
    # 收集所有工具结果
    results_text = "\n".join(
        f"- {r['tool']}: {r['result']}"
        for r in state["tool_results"]
    )

    # 用LLM生成自然语言回复
    prompt = (
        f"上下文：{state['context']}\n"
        f"工具结果：{results_text}\n"
        f"用户输入：{state['user_input']}\n"
        "请基于以上信息，生成自然、完整的回复。"
    )
    response = llm.invoke(prompt)
    return {"messages": [response]}

# ===== 构建完整工作流 =====
from langgraph.graph import StateGraph

workflow = StateGraph(AgentState)

# 添加所有节点
workflow.add_node("intent", intent_recognition_node)
workflow.add_node("context", context_management_node)
workflow.add_node("select", tool_selection_node)
workflow.add_node("execute", tool_execution_node)
workflow.add_node("memory", memory_governance_node)
workflow.add_node("respond", response_generation_node)

# 定义流转
workflow.set_entry_point("intent")
workflow.add_edge("intent", "context")
workflow.add_edge("context", "select")
workflow.add_edge("select", "execute")
workflow.add_edge("execute", "memory")
workflow.add_edge("memory", "respond")

# 编译+安全措施
app = workflow.compile(

✅ 全流程设计要点

① 模块化：每个节点职责单一——意图识别只做识别，不做执行；工具选择只做映射，不做调用。模块化让每个环节可独立测试、替换、升级。
② 结构化贯穿：从意图识别到工具结果，所有中间数据都用 Pydantic 模型约束。防止"中间格式崩塌"导致后续节点无法消费数据。
③ 记忆闭环：每次对话结束都更新记忆（STATE.md + 向量归档）。下次对话时，Agent 能从压缩上下文+向量召回中获得完整背景，而不是每次从零开始。
④ 安全兜底：工具执行失败→备选工具；验证失败→三层重试；意图未知→主动确认。每个环节都有降级方案，不会 crash。

16.11 企业级 Agent 架构设计

面试高频问题："如果让你做一个企业级 Agent，你会如何设计架构？"——这道题考的不是你会不会写 LangChain 代码，而是你能不能区分 Demo 和 Production。一个能跑 Demo 的架构，放到生产环境可能处处漏水。本节从 Demo 级 Agent 的五大缺陷出发，逐模块构建一个能扛住真实业务压力的企业级架构。

Demo 级 vs 企业级 Agent

⚠️ 面试踩坑：很多候选人一上来就画 LangChain 的链式调用图——那是在选框架，不是在设计架构。面试官问的是"企业级"，关键词是可靠性、可观测性、成本控制。先说清楚你解决了什么生产问题，再说你用什么技术实现。

八大核心模块

企业级 Agent 不是把 Demo 代码部署到服务器就完了——它需要在八个维度上都有独立的模块负责。每个模块只做一件事，但做到极致。下面是架构全景图：

🏗️ 八大核心模块职责与接口
模块 职责 核心接口 生产兜底
① Gateway层 请求接入、负载均衡、限流熔断、认证鉴权 handleRequest()、rateLimit()、authenticate() 熔断降级→返回默认响应
② Agent Engine层 意图识别、任务规划、执行调度、工具编排 classifyIntent()、planTasks()、executeStep() 意图未知→主动确认
③ Memory层 短期记忆（对话上下文）、长期记忆（向量DB）、工作记忆（任务状态） store()、retrieve()、compress()、evict() 记忆丢失→从摘要重建
④ Tool/Skill层 基础工具、组合流程、业务技能、MCP集成 register()、match()、invoke()、fallback() 工具挂→备选工具兜底
⑤ LLM Router层 模型选型路由、成本优化、缓存命中、降级回退 selectModel()、cacheQuery()、fallbackModel() 主模型挂→备模型接管
⑥ Context Engine层 上下文组装、压缩策略、缓存管理、token预算 assemble()、compress()、budget()、cache() 超预算→强制压缩
⑦ Safety层 权限控制、内容过滤、敏感操作审批、审计日志 checkPermission()、filterContent()、auditLog() 越权→阻断并记录
⑧ Observability层 运行日志、性能指标、成本追踪、异常告警 log()、trackMetrics()、alert()、reportCost() 旁路采集，不阻塞主流程

与第2章架构的关系

第2章（ch02）讲的是概念架构——Agent 的核心组件有哪些、各组件做什么、组件间如何协作。它回答的是"What"。
本章（ch16）讲的是生产实现架构——如何把这些概念组件变成可部署、可运维、可降级的工程系统。它回答的是"How"。

两者的关系：
• 第2章的"感知→决策→执行→记忆" → 本章拆分为 Gateway→Safety→Engine→Context→LLM→Tool→Memory
• 第2章的"LLM" → 本章细化为 LLM Router（带选型、缓存、降级）
• 第2章没有提到的 → 本章新增了 Observability（生产必须有观测）和 Context Engine（生产必须有上下文管理）

16.12 核心模块拆分与职责边界

面试高频问题："一个 Agent 系统应该拆分成哪些核心模块？每个模块分别负责什么？"——模块拆分不是"拆得越细越好"，而是拆得越清晰越好。每个模块的职责边界必须明确——只负责A，不负责B。如果边界模糊，模块间就会互相耦合，改一个模块牵连三个模块。

模块拆分三原则

📐 三大拆分原则
单一职责

每个模块只做一件事。Planner只负责规划，不负责执行；Memory只负责存取，不负责推理。就像公司里的岗位——HR不写代码，开发不做招聘。

接口隔离

模块间通过接口通信，不直接依赖内部实现。Planner不关心Executor用什么框架执行——它只关心Executor返回的result格式。换框架不影响Planner。

可替换性

任何模块都可以替换实现。LLM从GPT换到Claude不影响其他模块；Memory从Redis换到Milvus不影响Engine。关键：接口稳定，实现可变。

⚠️ 反面教材：把所有功能塞在一个"超级Agent"类里——意图识别、工具调用、记忆管理、安全校验都在一个3000行的类里。改个工具调用逻辑，可能影响意图识别的缓存。这种代码在Demo里能跑，在生产里是定时炸弹。

六大核心模块详解

职责边界卡片

🎯 Planner（规划器）

只负责：接收任务→生成执行计划→返回子任务序列
不负责：执行子任务、调用工具、存储结果
接口：plan(task: str) → List[SubTask]
降级：无法规划→返回单步执行（退化为ReAct）

⚡ Executor（执行器）

只负责：按计划执行→调用工具/模型→返回结果
不负责：规划下一步、决定用什么模型、管理记忆
接口：execute(plan: List[SubTask]) → ExecutionResult
降级：执行失败→重试1次→备选工具→报告失败

🧠 Memory（记忆器）

只负责：存储→索引→检索→压缩→淘汰
不负责：决定什么时候检索、检索结果怎么用
接口：store(key, value)、retrieve(query) → List[MemoryItem]
降级：向量DB挂→从Redis缓存召回；Redis挂→空记忆继续执行

🔧 ToolRouter（工具路由）

只负责：注册→匹配→调用→超时→降级
不负责：决定调用哪个工具（这是Executor的事）
接口：register(tool)、match(intent) → Tool、invoke(tool, params) → Result
降级：主工具超时→3秒后切备选工具→备选也挂→返回"无法执行"

🤖 LLMRouter（模型路由）

只负责：选型→路由→缓存→降级→计费
不负责：组装prompt（Context Engine的事）、解析LLM输出
接口：selectModel(complexity) → ModelName、invoke(model, prompt) → Response
降级：主模型挂→切备模型→切国产模型→切本地模型→返回默认响应

🔒 SafetyGuard（安全守卫）

只负责：权限检查→内容过滤→审计记录
不负责：执行业务逻辑、决定是否允许（只检查规则）
接口：checkPermission(user, action) → bool、filterContent(text) → SafeText
降级：安全服务挂→全阻断（宁可不可用，不可不安全）

模块间通信机制

📡 三种通信机制对比
机制 原理 优点 缺点 适用
消息队列 模块间通过异步消息传递，发布者→队列→订阅者 解耦彻底、可削峰、可重试 延迟高、调试难、需消息顺序保证 大规模
共享状态 模块间通过共享State对象通信，读写同一份数据 实时、简单、LangGraph原生方式 耦合度高、并发难、需状态锁 单Agent
事件驱动 模块发布事件，其他模块按需订阅，松耦合 灵活、可扩展、支持多消费者 事件顺序不保证、可能事件风暴 多Agent

实际选择：大多数 Agent 系统用混合模式——核心链路（意图→规划→执行）用共享状态（LangGraph的StateGraph），旁路模块（Observability、Safety）用事件驱动（异步采集不影响主流程），跨Agent协作用消息队列（Kafka/RabbitMQ）。

# ===== 六大核心模块接口定义 =====
from typing import Protocol, List, Optional
from dataclasses import dataclass

# ----- 数据结构定义 -----
@dataclass
class SubTask:
    task_id: str
    description: str
    tool_name: Optional[str]
    params: dict
    depends_on: List[str] = []      # 依赖哪些子任务

@dataclass
class ExecutionResult:
    task_id: str
    success: bool
    output: str
    tool_used: Optional[str]
    latency_ms: int

@dataclass
class MemoryItem:
    key: str
    value: str
    timestamp: float
    relevance_score: float

@dataclass
class SafeText:
    text: str
    is_safe: bool
    filtered_reason: Optional[str]

# ----- 模块接口定义 -----
class Planner(Protocol):
    """规划器：只负责生成执行计划"""
    def plan(self, task: str, context: dict) -> List[SubTask]:
        """接收任务，返回子任务序列"""
        ...

class Executor(Protocol):
    """执行器：只负责按计划执行"""
    def execute(self, plan: List[SubTask]) -> List[ExecutionResult]:
        """按计划执行，返回每个子任务的结果"""
        ...

class Memory(Protocol):
    """记忆器：只负责存储和检索"""
    def store(self, key: str, value: str, ttl: Optional[int] = None) -> None:
        """存储记忆"""
        ...
    def retrieve(self, query: str, top_k: int = 5) -> List[MemoryItem]:
        """检索相关记忆"""
        ...
    def compress(self) -> str:
        """压缩长期记忆"""
        ...

class ToolRouter(Protocol):
    """工具路由：只负责匹配和调用工具"""
    def register(self, name: str, tool: callable, fallback: Optional[callable] = None) -> None:
        """注册工具（含备选）"""
        ...
    def match(self, intent: str) -> Optional[str]:
        """根据意图匹配工具名"""
        ...
    def invoke(self, name: str, params: dict, timeout: int = 5) -> str:
        """调用工具，超时自动切备选"""
        ...

class LLMRouter(Protocol):
    """模型路由：只负责选型和调用模型"""
    def select_model(self, complexity: str) -> str:
        """根据复杂度选模型"""
        ...
    def invoke(self, model: str, prompt: str) -> str:
        """调用模型，失败自动降级"""
        ...

class SafetyGuard(Protocol):
    """安全守卫：只负责检查和过滤"""
    def check_permission(self, user: str, action: str) -> bool:
        """检查用户是否有权限执行该操作"""
        ...
    def filter_content(self, text: str) -> SafeText:
        """过滤敏感内容"""
        ...
    def audit_log(self, user: str, action: str, result: str) -> None:
        """记录审计日志"""
        ...

# ===== 关键设计点 =====
# 1. 每个Protocol只定义自己职责范围内的方法
# 2. 模块间不互相引用具体实现类，只依赖Protocol
# 3. 任何模块可以替换实现（如Memory从Redis→Milvus，只要实现Protocol即可）
# 4. 接口设计遵循"只进不出"原则：方法返回结果，不修改共享状态
📋 八股总结 — 面试高频考点
Q1: 什么场景需要 Agent？什么场景不需要？
需要 Agent：任务需要自主决策——面对不确定环境，自己决定下一步做什么。如：研究分析、代码生成、复杂客服。
不需要 Agent：固定流程的任务。如：ETL 管道、定时报表、固定步骤的工单处理。用规则引擎/工作流即可。

判断标准：流程能否预先确定？能→不用 Agent；不能→考虑 Agent。
Q2: 从0到1构建 Agent 的五个阶段是什么？
① 需求分析：明确要解决什么问题，确定是否需要 Agent、需要什么能力。
② 架构设计：选择框架（LangGraph/AutoGen/CrewAI/Dify）、选择模型（GPT-4o/通义千问）、设计 State 和工具。
③ 开发实现：定义 State、工具、节点，构建工作流，加入安全措施。
④ 测试评估：跑测试集，评估任务完成率、准确性、安全性。
⑤ 部署上线：容器化、负载均衡、监控告警、成本控制。
Q3: 选择 LLM 模型时需要考虑哪些因素？
① 推理能力：复杂推理用 GPT-4o/Claude 3.5，简单对话用 mini 版本
② 成本：大规模用便宜模型（GPT-4o-mini/通义Turbo），关键决策用强模型
③ 延迟：实时场景需要快速响应（首token < 1s）
④ 数据安全：敏感数据用本地部署（Llama/Qwen）
⑤ 语言：中文场景考虑国产模型（通义千问/DeepSeek）
⑥ 工具调用能力：Function Calling 的准确率和稳定性
Q4: Agent 生产部署需要哪些安全措施？
① API Key 管理：用 Vault/AWS Secrets Manager，不硬编码
② 速率限制：防止滥用和成本失控
③ 输入过滤：防止提示注入攻击
④ 输出审核：防止敏感信息泄露
⑤ 工具白名单：限制 Agent 可调用的工具
⑥ 人工干预：危险操作前暂停确认
⑦ 预算上限：每日 Token 消耗上限
Q5: 怎么控制 Agent 的运行成本？
① 模型降级：简单步骤用 mini 模型，复杂步骤用强模型
② 缓存：常用查询结果缓存，避免重复调用 LLM
③ 步数限制：限制最大循环次数，防止无限执行
④ Prompt 精简：减少不必要的上下文，降低 Token 消耗
⑤ 批量处理：合并多个请求，减少 API 调用次数
⑥ 监控告警：实时监控 Token 消耗，超预算告警
Q6: 意图识别的三种类型是什么？如何处理复合意图？
显式意图：用户直接表达目标（"帮我查天气"）→直接映射工具
隐式意图：用户描述场景而非需求（"明天去上海出差"）→需推断实际需求
复合意图：一句话包含多个子目标（"出差帮我安排"）→需拆解为子意图序列

复合意图处理：① 拆解为子意图 ② 分析依赖关系（哪些可并行/串行） ③ 按依赖排序执行。关键：订机票依赖确定日期，推荐酒店依赖知道目的地——不能无序执行。
Q7: Context Engineering 的三大策略是什么？各解决什么问题？
① 压缩策略：解决对话历史过长的问题。不是简单截断，而是智能摘要+选择性保留关键信息。核心：100 tokens精准上下文 > 10000 tokens无关废话。
② 结构化笔记：解决对话历史信息密度低的问题。让Agent主动维护STATE.md/TODO.md，比对话历史Token消耗低10倍，检索效率高，可更新。
③ Multi-Agent拆分：解决单Agent Prompt膨胀的问题。拆为多个专业Agent，每个只关注自己的领域，上下文自然精简，注意力集中，输出精准。
Q8: 什么是 Context Rot？如何治理？
Context Rot：长时间运行后上下文腐烂——信息过期、重复、噪声堆积，导致Agent性能持续退化。

三大症状：① 信息过期（旧决策不再有效）② 信息重复（同一件事反复确认）③ 噪声堆积（寒暄+错误尝试淹没关键信息）

四步治理：① 锁定核心（不可丢弃的决策+偏好）② 确保完整性（摘要覆盖所有关键话题）③ 保持连贯性（按时间线组织）④ 添加系统提示（告知Agent这是压缩后的历史）。配合滑动窗口（保留近N轮）+向量库归档（超长对话按需召回）。
Q9: 企业级 Agent 和 Demo 级 Agent 的核心区别是什么？
从三个维度分析：
① 可靠性：Demo级单进程运行，崩了就没了；企业级多实例+故障转移+熔断降级，任何一个环节挂了都有兜底方案。
② 可观测性：Demo级出了问题靠猜；企业级全链路追踪（Langfuse/LangSmith）+性能指标（Prometheus）+成本追踪+异常告警，任何异常都能定位到具体环节。
③ 成本控制：Demo级所有步骤用最强模型，Token费用不可控；企业级LLM Router根据复杂度选模型、缓存常用查询、日预算上限+告警，成本可控可预测。

一句话：Demo级追求"能跑"，企业级追求"能跑+能扛+能降+能查+能控"。
Q10: 为什么 Agent 系统需要模块化拆分？所有功能塞在一个模块里会带来什么问题？
三个严重问题：
① 耦合爆炸：改一个功能牵连三个功能。比如改工具调用逻辑，可能影响意图识别的缓存。3000行的"超级Agent"类，任何修改都是高风险。
② 无法降级：所有功能在一起，无法单独替换某个模块的实现。LLM从GPT换到Claude？需要改整个类。Memory从Redis换到Milvus？又要改整个类。
③ 无法观测：没有清晰的模块边界，出了问题无法定位是哪个环节的bug。日志里只有一行"Agent执行失败"，不知道是意图识别错了还是工具调用挂了。

模块化的核心价值：每个模块职责单一、接口稳定、实现可替换、降级独立。改一个不影响其他，挂一个不影响整体。
Q11: 请设计一个企业级 Agent 架构，说明八大核心模块的职责和接口。
八大核心模块：
① Gateway层：请求接入、负载均衡、限流熔断、认证鉴权。接口：handleRequest()、rateLimit()、authenticate()。兜底：熔断→返回默认响应。
② Agent Engine层：意图识别、任务规划、执行调度、工具编排。接口：classifyIntent()、planTasks()、executeStep()。兜底：意图未知→主动确认。
③ Memory层：短期记忆（对话上下文）、长期记忆（向量DB）、工作记忆（任务状态）。接口：store()、retrieve()、compress()、evict()。兜底：记忆丢失→从摘要重建。
④ Tool/Skill层：基础工具、组合流程、业务技能、MCP集成。接口：register()、match()、invoke()、fallback()。兜底：工具挂→备选工具。
⑤ LLM Router层：模型选型路由、成本优化、缓存命中、降级回退。接口：selectModel()、cacheQuery()、fallbackModel()。兜底：主模型挂→备模型接管。
⑥ Context Engine层：上下文组装、压缩策略、缓存管理、token预算。接口：assemble()、compress()、budget()、cache()。兜底：超预算→强制压缩。
⑦ Safety层：权限控制、内容过滤、敏感操作审批、审计日志。接口：checkPermission()、filterContent()、auditLog()。兜底：越权→阻断并记录。
⑧ Observability层：运行日志、性能指标、成本追踪、异常告警。接口：log()、trackMetrics()、alert()、reportCost()。兜底：旁路采集，不阻塞主流程。

架构设计要点：每个模块只做一件事、通过接口通信、任何模块可替换实现、每个环节都有降级方案。与第2章概念架构的关系：第2章回答"What"（Agent有哪些组件），本章回答"How"（如何把组件变成生产系统）。
